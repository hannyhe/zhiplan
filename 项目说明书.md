# 致知计划内容收益 — 日报自动更新系统

## 概述

本系统每日自动从 `致知计划内容收益.xls`（源数据文件，模糊匹配文件名中的"致知计划内容收益"）读取收益数据，在对应月份的月报工作簿（如 `2607.xlsx`）中创建当日新工作表（如 0705），写入原始数据与计算公式，完成跨日对比统计。目标工作簿根据当前日期自动确定：3 日及以后使用当月文件，1‑2 日使用上月文件。每月 2 日自动拆分新月度文件（复制最后 2 表，断开跨月链接）。

## 数据结构（每 sheet 15 列）

| 列 | 字段 | 说明 |
|----|------|------|
| **A** | 问题标题 | 原始数据 |
| **B** | 类型 | 想法/回答/文章 |
| **C** | 发布时间 | yyyy/M/d |
| **D** | 日内阅读/观看量 | 原始数据 |
| **E** | 日内收益 | 原始数据 |
| **F** | 累计阅读/观看量 | 原始数据，用于 J 列计算 |
| **G** | 累计收益 | 原始数据，用于 K 列计算 |
| **H** | `=ROUND(E/D,2)` | 当日单次阅读收益 |
| **I** | `=ROUND(G/F,2)` | 累计单次阅读收益 |
| **J** | **阅读差值** | `=F - VLOOKUP(A, 前日!F, ...)` 三层回退公式 |
| **K** | **收益差值** | `=G - VLOOKUP(A, 前日!G, ...)` 三层回退公式 |
| **L** | `=IFERROR(ROUND(K/J,2),)` | 当日单次收益（保留 2 位小数） |
| **M** | 前日 M（文本） | 直接从昨日 sheet 对应行拷贝 M 列值 |
| **N** | 前日 N（文本） | 同上，N 列 |
| **O** | 前日 O（文本） | 同上，O 列 |

### 核心公式

**J 列（阅读差值）** — 累计阅读差，三层回退：
```
=F{rr} - IFERROR(
    VLOOKUP($A{rr}, '前日'!$A$1:G{lastRow}, 6, FALSE),         ← 第 1 层：按 A 列精确匹配昨日 F
    IFERROR(
        VLOOKUP($C{rr}, '前日'!$C$1:G{lastRow}, 4, FALSE),     ← 第 2 层：按 C 列（发布时间）匹配昨日 D
        F{rr} - D{rr}                                           ← 第 3 层：回退为当日（F - D）
    )
)
```

**K 列（收益差值）** — 累计收益差，三层回退：
```
=G{rr} - IFERROR(
    VLOOKUP($A{rr}, '前日'!$A$1:H{lastRow}, 7, FALSE),         ← 第 1 层：按 A 列精确匹配昨日 G
    IFERROR(
        VLOOKUP($C{rr}, '前日'!$C$1:H{lastRow}, 5, FALSE),     ← 第 2 层：按 C 列匹配昨日 E
        G{rr} - E{rr}                                           ← 第 3 层：回退为当日（G - E）
    )
)
```

## 匹配逻辑（3 层行匹配，用于 M/N/O 列）

由于 M/N/O 需要从昨日 sheet 拷贝文本数据，而 Excel 公式无法实现模糊匹配，因此在 PowerShell 中预计算匹配行索引：

| 层级 | 算法 | 阈值 |
|------|------|------|
| **1a** | A 列精确匹配（哈希表） | 100% |
| **1b** | A 列 Levenshtein 模糊匹配 | ≥ 0.65 |
| **2** | 按 B 列（类型）筛选 → C 列（发布时间）精确匹配 | — |
| **3** | 按 C 列筛选 → B 列精确 → A 列模糊验证 | ≥ 0.3 |
| **失败** | 全新问题，M/N/O 置空 | — |

模糊匹配采用 Levenshtein 编辑距离 → `1 - (编辑距离 / 最长串长度)`。

## 关键技术决策

### 1. J/K 公式使用累计列（F/G），而非日列（D/E）

原版 `AutoUpdate.ps1` 错误使用 `=IF(M=0, D, D-M)`，把 M 当作昨日 D 的硬编码值，但 M 列是批注文本，且 D 是日数据而非累计数据。

**修正**：J = 今日累计 F − 昨日累计 F（VLOOKUP 三层回退），与 `2505.xlsx` 样本公式一致。

### 2. M/N/O 来源为昨日 M/N/O（文本拷贝），而非昨日 D/E/F

原版将 M/N/O 写为 `='前日'!D/E/F{n}` 公式引用昨日原始数据。

**修正**：直接从 `$pRaw[mr,13..15]` 取值写入，内容为前日 M/N/O 列的文本/批注。

### 3. 两阶段写入策略 + 批量 Range 赋值避免 COM 冲突

Excel COM 在大量单格写入后调用 `Save()` 时，容易抛出 `命令不可用`（0x800AC472）错误。

**解决**：
- 第 1 阶段：写入 A-G 原始值 + M/N/O 文本（Cell.Item 单格写入）
- 第 2 阶段：H-L 公式用 `Range("H2:H50").Formula = $array` 批量赋值
- 5 个公式列各一次 COM 调用，共 5 次，而非 5×49=245 次

### 4. 日期命名规则

自动识别当前工作簿中最高 4 位数字 sheet 名（如 0704），+1 得到新 sheet 名（0705）。新 sheet 自动置末。

### 5. 自动文件名与月度拆分

- **源文件自动检测**：通过 `Get-ChildItem | Where-Object { $_.BaseName -match "致知计划内容收益" }` 模糊匹配文件名，无需手动指定
- **目标文件自动确定**：当前日期 ≥ 3 日 → 当月文件 `{YYMM}.xlsx`；1‑2 日 → 上月文件
- **月度拆分**：每月 2 日运行完每日更新后，自动复制当前文件最后 2 个数字 sheet 到新的月度文件（如 `2607.xlsx` → `2608.xlsx`），第一个 sheet 转换为纯值（断开跨月链接），第二个 sheet 保留公式

### 6. 外部引用修复

复制 sheet 到新文件时，Excel 会自动将 sheet 间引用转为外部文件引用（`'[2607.xlsx]0701'!`）。脚本在保存前遍历公式数组，批量替换为本地引用（`'0701'!`），确保新月度文件独立可用。

### 7. 中文编码

所有中文字符串使用 `[char]0xXXXX` + `` `${var} `` 引用写法，避免 PowerShell 编码损坏。

## 运行方式

脚本已实现全自动路径检测，无需手动指定文件名。

```powershell
# 终止残留 Excel 进程
Get-Process -Name "*excel*" | Stop-Process -Force
Start-Sleep -Seconds 15

# 直接运行（自动检测源文件和目标文件）
powershell -NoProfile -ExecutionPolicy Bypass -File C:\Users\coldt\Desktop\update0703.ps1
```

如需修改数据目录，编辑脚本顶部的 `$dataDir` 配置变量：

```powershell
$dataDir = "G:\me\致知收益\2025"  # 修改此路径即可切换目录
```

## 与旧版（AutoUpdate.ps1）差异对照

| 项目 | 旧版（错误） | 修正版 |
|------|-------------|--------|
| **J 公式** | `=IF(M=0,D,D-M)` 用日数据 D | `=F - VLOOKUP(前日!F, ...)` 用累计数据 F |
| **K 公式** | `=IF(N=0,E,E-N)` 用日数据 E | `=G - VLOOKUP(前日!G, ...)` 用累计数据 G |
| **M 列内容** | `=前日!D`（公式引用昨日日数据） | 直接拷贝昨日 M 列文本 |
| **N 列内容** | `=前日!E`（公式引用昨日日数据） | 直接拷贝昨日 N 列文本 |
| **O 列内容** | `=前日!F`（公式引用昨日日数据） | 直接拷贝昨日 O 列文本 |
| **L 公式** | `=IF(J=0, 0, ROUND(K/J,2))` | `=IFERROR(ROUND(K/J,2),)` |
| **新题 MNO** | 硬编码 0 | 空文本 "" |
| **匹配算法** | 仅精确匹配 A 列 | 3 层匹配（精确→模糊→B→C） |
| **公式写入** | 逐格循环（245 次 COM） | 批量 Range 赋值（5 次 COM） |
| **L/合计四舍五入** | 仅 L 行内 ROUND | L 与合计 K 均 ROUND 到 2 位小数 |

## 已知问题 / 待改进
