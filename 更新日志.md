# 更新日志

## v2.3 — J/K 公式改用 PowerShell 模糊匹配结果，彻底解决标题不一致导致的 VLOOKUP 失败 (2026-07-11)

### 1. 变更原因

v2.1 移除 C 列回退后，A 列精确匹配失败直接回退到 `F-D`/`G-E`。但若 A 列因 emoji 等字符前后日不一致（如 `✨​` vs `??`），Excel VLOOKUP 精确匹配必然失败。即使 v2.2 加入了 NormalizeTitle 规范化，也仍需过渡期逐步修正前日数据。

**根本解决方案**：让 J/K 公式直接引用 PowerShell 模糊匹配确定的行列，不再依赖 Excel VLOOKUP。

### 2. 公式变更

```excel
旧: =F-IFERROR(VLOOKUP($A, '前日'!$A:G, 6, FALSE), F-D)
新: =F-IFERROR('前日'!F{匹配行}, F-D)
```

匹配行号由 PowerShell 的 3 层匹配（精确→Levenshtein≥0.65→B→C→新）提前计算好，公式直接引用编号。核心公式文件位置：`update0703.ps1:187-192`。

### 3. 无匹配时处理

- 无匹配直接回退 `=F-D` / `=G-E`（旧版 VLOOKUP 失败时的行为保持一致，但公式结构更简单）

### 4. 效果

- A 列字符差异不再影响 J/K 计算
- 公式更短、计算更快
- 消除 VLOOKUP 对前日数据格式的依赖，一次匹配永不偏移

## v2.2 — 标题规范化，修复 emoji 导致匹配失败 (2026-07-11)

### 1. 新增 NormalizeTitle 规范化函数

源数据标题中可能包含 emoji、零宽空格等字符，不同日期同一内容的标题字符不一致，导致精确匹配失败。

新增 `NormalizeTitle` 函数，统一规范化规则：
- 剔除零宽空格（U+200B-U+200D、U+FEFF、U+00AD）
- 代理对（emoji 等高码位字符）替换为 `?`
- ASCII `?` 字符也参与折叠（连续 `?` → 单个 `?`）
- 非 ASCII、非 CJK、非标准标点的字符统一替换为 `?`
- 连续被替换的 `?` 自动合并

应用于：
- 源数据读取时（第 93 行）→ 写表时使用规范化标题
- 前日表读取时（第 129 行）→ 匹配键使用规范化标题
- 前日表 A 列回写（第 138-142 行）→ 确保 VLOOKUP 跨日一致性

### 2. 匹配效果提升

0712 创建测试（47 行源数据，前日 0711）：
- **规范化前**：39 精确 + 1 模糊 + 0 B + 0 C + 7 全新（0711 运行）
- **规范化后**：45 精确 + 0 模糊 + 0 B + 0 C + 2 全新（0712 运行）
- 前日表 5 行标题被规范化回写
- 受影响行的 J/K 从回退值恢复为正确的累计差值

### 3. 前日表回写机制

每次运行自动检测前日表 A 列标题是否需要规范化，若需要则直接更新单元格值。确保新旧版本交替时 VLOOKUP 公式能正确匹配，避免单次过渡期偏差。

## v2.1 — 公式精简与源文件验证 (2026-07-10)

### 1. J/K 列公式移除 C 列第二层回退

J 列（阅读差值）与 K 列（收益差值）的三层回退公式中，按 C 列（发布时间）匹配的第二层已移除，从三层变为两层：

- **J 列**：`=F - IFERROR(VLOOKUP(A, 前日!F, 6, FALSE), F - D)`
- **K 列**：`=G - IFERROR(VLOOKUP(A, 前日!G, 7, FALSE), G - E)`

第一层 VLOOKUP 按 A 列（问题标题）精确匹配失败后，直接回退到当日累计差值（F-D / G-E），不再尝试按 C 列匹配。

### 2. 源数据文件创建时间验证

源数据文件检测增加创建时间过滤，仅接受**当天生成**的文件：

- `$_.CreationTime.Date -eq (Get-Date).Date`
- 若无当天文件，脚本立即终止并抛出错误
- 避免误用陈旧或跨日遗留的源数据文件

### 3. 测试验证（2026-07-10）

- 源数据：47 行
- 匹配结果：39 精确 + 1 模糊 + 0 B + 0 C + 7 全新
- 公式验证：J/K 无 C 列回退 ✓ | L 列正常 ✓ | 保存 OK

## v2.0 — 通用化改造 (2026-07-05)

### 1. 目标文件名自动生成

根据当前日期自动确定所属月份的工作簿文件：

- **每月 3 日至下月 2 日**：使用**本月**文件名（如 7 月 3 日～8 月 2 日使用 `2607.xlsx`）
- **每月 1～2 日**：使用**上月**文件名（如 7 月 1～2 日仍使用 `2606.xlsx`）

年份前缀（`26`）从日期自动推算，跨年时（如 1 月 2 日使用上年 12 月文件）自动处理。

### 2. 跨月文件拆分（每月 2 日）

每月 2 日更新完成后，自动执行月文件拆分：

1. 将当前工作簿中**最后两张表**（如 `0701`、`0702`）复制到新文件
2. 新文件命名为**当月**文件名（如 `2607.xlsx`）
3. **断开与上月的数据链接**：
   - 首张表（`0701`）所有公式转为静态值（因其引用的上月表 `0630` 不在新文件中）
   - 第二张表（`0702`）保留公式（其引用的 `0701` 表存在于新文件中）
4. 删除新文件中的默认空白表

### 3. 源数据文件模糊匹配

不再要求文件名完全固定为"致知计划内容收益.xls"，只要文件名中包含"致知计划内容收益"即可自动匹配。支持以下情况：

- `致知计划内容收益.xls` ✓
- `致知计划内容收益2407.xls` ✓
- `致知计划内容收益240828.xls` ✓

### 4. 路径配置化

移除了脚本中的硬编码路径，统一通过 `$dataDir` 变量配置工作目录。
