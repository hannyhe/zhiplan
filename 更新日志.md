# 更新日志

## v2.1 — 公式精简与源文件验证 (2026-07-10)

### 1. J/K 列公式移除 C 列第二层回退

J 列（阅读差值）与 K 列（收益差值）的三层回退公式中，按 C 列（发布时间）匹配的第二层已移除，从三层变为两层：

- **J 列**：`=F - IFERROR(VLOOKUP(A, 前日!F, 6, FALSE), F - D)`
- **K 列**：`=G - IFERROR(VLOOKUP(A, 前日!G, 7, FALSE), G - E)`

第一层 VLOOKUP 按 A 列（问题标题）精确匹配失败后，直接回退到当日累计差值（F-D / G-E），不再尝试按 C 列匹配。

### 2. 源数据文件创建时间验证

源数据文件检测增加创建时间过滤，仅接受**当天生成**的文件：

- `$_.CreationTime.Date -eq (Get-Date).Date`
- 若无当天文件，脚本立即终止并抛出错误
- 避免误用陈旧或跨日遗留的源数据文件

### 3. 测试验证（2026-07-10）

- 源数据：47 行
- 匹配结果：39 精确 + 1 模糊 + 0 B + 0 C + 7 全新
- 公式验证：J/K 无 C 列回退 ✓ | L 列正常 ✓ | 保存 OK

## v2.0 — 通用化改造 (2026-07-05)

### 1. 目标文件名自动生成

根据当前日期自动确定所属月份的工作簿文件：

- **每月 3 日至下月 2 日**：使用**本月**文件名（如 7 月 3 日～8 月 2 日使用 `2607.xlsx`）
- **每月 1～2 日**：使用**上月**文件名（如 7 月 1～2 日仍使用 `2606.xlsx`）

年份前缀（`26`）从日期自动推算，跨年时（如 1 月 2 日使用上年 12 月文件）自动处理。

### 2. 跨月文件拆分（每月 2 日）

每月 2 日更新完成后，自动执行月文件拆分：

1. 将当前工作簿中**最后两张表**（如 `0701`、`0702`）复制到新文件
2. 新文件命名为**当月**文件名（如 `2607.xlsx`）
3. **断开与上月的数据链接**：
   - 首张表（`0701`）所有公式转为静态值（因其引用的上月表 `0630` 不在新文件中）
   - 第二张表（`0702`）保留公式（其引用的 `0701` 表存在于新文件中）
4. 删除新文件中的默认空白表

### 3. 源数据文件模糊匹配

不再要求文件名完全固定为"致知计划内容收益.xls"，只要文件名中包含"致知计划内容收益"即可自动匹配。支持以下情况：

- `致知计划内容收益.xls` ✓
- `致知计划内容收益2407.xls` ✓
- `致知计划内容收益240828.xls` ✓

### 4. 路径配置化

移除了脚本中的硬编码路径，统一通过 `$dataDir` 变量配置工作目录。
